# ADR 0001: Record Architecture Decisions

## Status
Accepted

## Date
2026-02-27

## Context
We need to record significant architecture decisions made during the project.

## Decision
We will use Architecture Decision Records (ADRs) stored in `docs/adr/`.

Each ADR is a short text file describing a single significant decision.

## Template for new ADRs
```
# ADR XXXX: Title

## Status
[Proposed | Accepted | Deprecated | Superseded by ADR-XXXX]

## Date
YYYY-MM-DD

## Context
What is the issue that motivates this decision?

## Decision
What is the decision that was made?

## Consequences
What are the resulting context and trade-offs?
```

## Consequences
Developers can understand why technical decisions were made, even long after the fact.
